## How to read this

Each case follows the same structure:
- **Context** — what the project was and what was at stake
- **My role** — what I specifically did (ops, not engineering)
- **Process** — how I worked through it
- **Outcome** — what changed

The process files (`uat-process.md`, `issue-prioritization.md`, `delivery-framework.md`) are reusable frameworks I've applied across projects.
